# Comparison of Neural Network Surrogate and Physics Model

This document explains how the **Machine Learning (ML) Surrogate Model** is compared against the **Actual Physics Model** in the Mass Transfer Digital Twin project.

## 1. The Two Models

### Physics Model (The "Ground Truth")
*   **Location**: `mass_transfer/core/crosscurrent.py`
*   **Function**: `solve_crosscurrent`
*   **Description**: This is a rigorous mathematical solver that uses equilibrium data (tie-lines) and mass balance equations to calculate the stage-by-stage extraction of a solute. It is accurate but computationally intensive, especially when running thousands of simulations for optimization or data generation.

### NN Surrogate Model (The "Fast Proxy")
*   **Location**: `mass_transfer/ml/neural_net.py`
*   **Architecture**: A Feedforward Artificial Neural Network (ANN) built with **PyTorch**.
*   **Description**: Once trained on data generated by the physics model, this model "learns" the relationship between input parameters (number of stages, solvent flow, feed composition) and the output (percent removal). It provides near-instantaneous results.

---

## 2. Comparison Workflow

The comparison logic is implemented in the **Surrogate Model** tab of the GUI (`mass_transfer/gui/surrogate_tab.py`).

### Step A: Data Generation for Comparison
The system runs a background worker (`NNComparisonWorker`) to generate fresh test data that the neural network has not seen during training. This ensures a fair evaluation of its performance.

There are two primary ways the comparison is conducted:
1.  **Scatter Mode (Random Sampling)**: Uses **Latin Hypercube Sampling (LHS)** to pick random points across the entire operating range (varying stages, solvent, and acid concentration simultaneously).
2.  **Sweep Mode (Parametric Study)**: Varies exactly one parameter (e.g., Solvent Flow) while keeping all other inputs fixed. This shows how well the NN captures the "trend" of the physical process.

### Step B: Simultaneous Prediction
For every test point generated in Step A:
1.  The **Physics Model** calculates the "Actual" result.
2.  The **NN Surrogate** calculates the "Predicted" result using the `predict()` function.

### Step C: Metric Calculation
The system calculates several statistical metrics to quantify the accuracy of the surrogate:
*   **R² (Coefficient of Determination)**: Tells us how much of the physics model's behavior is captured by the NN. A value of 1.00 is a perfect match.
*   **MAE (Mean Absolute Error)**: The average difference (in percentage points) between the NN and the solver.
*   **RMSE (Root Mean Square Error)**: Similar to MAE but penalizes larger errors more heavily.
*   **Max Error**: The single largest mistake the NN made across all test points.

## 3. Visualization and Methodology

The system provides two distinct ways to visualize and evaluate the surrogate model's accuracy. Both methods use the **Total Solute Removal (%)** as the primary output variable being compared.

### Method 1: Scatter Comparison (Domain-Wide Accuracy)
*   **Methodology**: Uses **Latin Hypercube Sampling (LHS)** to generate a set of $N$ random test points (defaulting to 50 or 100). LHS ensures that the test points are spread evenly across the entire multi-dimensional input space, rather than clustering in one area.
*   **Input Variables**:
    *   `Number of Stages` (1 to 15)
    *   `Solvent per Stage` (100 to 5000 kg)
    *   `Feed Acid Composition` (5% to 45%)
*   **Visualizations**:
    1.  **Parity Plot (Predicted vs. Actual)**: Plots every test point on a graph where the X-axis is the "True" solver result and the Y-axis is the NN prediction. A perfect model would have all points on the $y = x$ line. Points are colored by their absolute error (Green = low, Red = high).
    2.  **Error Histogram**: A frequency distribution of the residuals ($y_{pred} - y_{true}$). A narrow bell curve centered at zero indicates a well-calibrated model with minimal bias.

### Method 2: Parametric Sweep Comparison (Trend Validation)
*   **Methodology**: This method keeps two input variables fixed at user-defined values and "sweeps" the third variable across its entire range. This is used to verify if the NN correctly captures the **physical trends** (sensitivity) of the process.
*   **Variables**:
    *   **Sweep Variable**: Any one of `Stages`, `Solvent Flow`, or `Feed Acid %`.
    *   **Fixed Variables**: The remaining two are held constant at the values set in the GUI's "Comparison" sub-tab.
*   **Visualizations**:
    1.  **Dual Line Chart**: Displays two overlapping lines: a solid blue line for the Physics Solver and a dashed red line for the NN Surrogate. It shows how `% Removal` changes as the sweep variable increases.
    2.  **Residual Area Chart**: A sub-plot at the bottom showing the absolute error ($|Error|$) across the sweep range. This highlights where the model is most/least accurate (e.g., the NN might be very accurate at low solvent flows but lose precision at extremely high flows).

---

## 4. Key Metrics Compared
Regardless of the visualization method, the following metrics are calculated:
*   **MAE (Mean Absolute Error)**: Average deviation in removal percentage.
*   **Max Error**: The "worst-case" prediction failure in the test set.
*   **R² (Scatter only)**: The global correlation coefficient.
