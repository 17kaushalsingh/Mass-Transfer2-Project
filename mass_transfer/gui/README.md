# Graphical User Interface (`mass_transfer/gui/`)

This directory contains the PyQt6-based desktop application that serves as the interactive frontend for the mass transfer digital twin. 

## Purpose
The GUI provides an intuitive, user-friendly environment for chemical engineering students, researchers, and process engineers to interact with the underlying complex mathematical solvers without writing any code. It orchestrates the flow of data between the core solvers, the visualization engine, and the machine learning surrogate models.

## Application Structure

The application is structured around a central `MainWindow` that houses multiple specialized functional tabs:

*   **`main_window.py` / `__main__.py`**: Entry points that initialize the Qt application, apply the global theme, and construct the main tabbed interface.
*   **`data_input_tab.py`**: Allows users to load ternary equilibrium data (from JSON files), inspect it in a tabular format, and trigger the core equilibrium fitting routines.
*   **`simulation_tab.py`**: The control center for running extraction models. Users can configure feed parameters, stage counts, and solvent flows, then execute crosscurrent or countercurrent solvers.
*   **`comparison_tab.py`**: A specialized view designed to run and compare both crosscurrent and countercurrent configurations side-by-side for a given set of operating conditions.
*   **`heatmap_tab.py`**: Renders dynamic, interactive heatmaps of stage-by-stage compositions, flow rates, and percentage removals generated by the `viz` module.
*   **`surrogate_tab.py`**: Interfaces with the `ml` module to generate synthetic datasets, train the neural network surrogate, run near-instantaneous predictions, and display 3D response surfaces.
*   **`animation_tab.py`**: Displays dynamic GIFs and stepping animations that visually explain the physical stage-to-stage extraction process.
*   **`theme.py` & `ui_helpers.py`**: Shared UI utilities for consistent styling, progress bars, layout management, and standardized widget creation across the entire application.

## Integration & Concurrency
The GUI strictly handles presentation and user input logic. Any computationally heavy tasks (like solving nonlinear systems, generating synthetic data, or training neural networks) are dispatched to the backend `core` and `ml` packages. These heavy tasks utilize threading and worker pools to ensure the graphical interface remains responsive during long calculations.
